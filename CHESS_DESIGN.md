# KC Gaming Chess 2026 - 设计文档

## 📋 项目概述

本项目是KC Gaming Chess的全面重构，采用2026年全新赛制，所有比赛在LiChess平台线上举行。

## 🎨 设计理念

### 视觉设计
- **现代化风格**：采用深色主题，融合国际象棋元素
- **配色方案**：
  - 主色：金色 (#ffd700) - 象征荣耀与竞技
  - 次色：青蓝色 (#00d4ff) - 代表Master赛事
  - 背景：深蓝黑渐变 - 营造专业氛围
- **棋盘元素**：背景使用棋盘网格，点缀国际象棋棋子符号
- **动画效果**：流畅的渐变动画、悬停效果、滚动淡入

### 用户体验
- **响应式设计**：完美适配桌面、平板、手机
- **清晰导航**：顶部固定导航栏，年份切换按钮
- **信息层次**：从赛季概览 → 赛制说明 → 规则 → 赛程 → 排行榜
- **交互反馈**：按钮悬停、卡片点击、弹窗详情

## 📊 积分系统

### 年度积分榜
- **统计周期**：自然年度（1月1日 - 12月31日）
- **排名依据**：
  1. 总积分（降序）
  2. 冠军数（降序）
  3. 亚军数（降序）
  4. 季军数（降序）

### 自动计算
- 使用JavaScript实时计算积分
- 从比赛results字段自动提取数据
- 支持历史记录查看
- 点击选手可查看详细信息

### 成就系统
- **Grand Slam**：赢得所有4场Master赛事
- **Elite Master**：赢得6场或以上Elite赛事
- **Consistency Award**：参加全年所有16场比赛
- **Perfect Record**：全胜战绩（至少5场）

## 📝 维护指南

### 添加新比赛
1. 打开 `_data/chess.yml`
2. 在 `tournaments` 数组中添加新条目
3. 设置 `status: "upcoming"`
4. 填写日期、时间、链接

### 更新比赛结果
1. 修改对应比赛的 `status: "completed"`
2. 在 `results` 数组中添加获奖者
3. 格式：`{ position: "🥇 冠军", name: "选手姓名" }`
4. 排行榜会自动更新

### 新增赛季
1. 创建新的 `_data/chess_20XX.yml`
2. 创建对应的 `_layouts/chess_20XX.html`
3. 创建页面入口 `chess_20XX.md`
4. 更新年份切换按钮

## 🐛 已知问题与解决方案

### 问题1：Jekyll不支持动态JSON
**解决方案**：使用Liquid的 `jsonify` 过滤器将YAML转换为JSON传递给JavaScript

### 问题2：中文名称处理
**解决方案**：积分系统会自动提取英文名，忽略昵称部分

### 问题3：时区问题
**解决方案**：所有时间使用本地时区，无需转换
